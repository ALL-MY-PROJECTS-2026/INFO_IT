React 앱을 만들 때 가장 먼저 마주치는 갈림길이 **렌더링 방식**입니다.
같은 화면을 그리더라도, 어디서 HTML을 만드느냐에 따라 SEO 결과가 크게 달라집니다.

## 세 가지 렌더링 방식

### CSR (Client-Side Rendering)

브라우저가 빈 HTML을 받은 뒤 JavaScript로 화면을 그립니다.
Create React App의 기본 방식입니다.

```html
<!-- 크롤러가 처음 받는 HTML -->
<div id="root"></div>
```

문제는 검색 크롤러가 **처음에 빈 페이지**를 본다는 것입니다.
Googlebot은 JS를 실행하긴 하지만, 렌더링이 2차 큐로 밀려 색인이 지연되거나 실패할 수 있습니다.

### SSR (Server-Side Rendering)

요청마다 서버가 완성된 HTML을 생성해 내려줍니다. 항상 최신이지만 서버가 필요합니다.

### SSG (Static Site Generation)

**빌드 시점에** 각 페이지를 정적 HTML로 미리 생성합니다.
콘텐츠가 자주 바뀌지 않는 블로그에 가장 잘 맞습니다.

## 비교 정리

| 방식 | 초기 HTML | SEO | 서버 필요 | 블로그 적합도 |
|------|-----------|-----|-----------|----------------|
| CSR  | 비어 있음 | 불리 | X | 낮음 |
| SSR  | 완성됨 | 좋음 | O | 중간 |
| SSG  | 완성됨 | 좋음 | X | **높음** |

## 결론

콘텐츠 중심 블로그라면 **SSG**가 정답에 가깝습니다.
검색·광고 크롤러가 완성된 HTML을 즉시 받고, CDN에 정적 배포할 수 있어 성능·비용 모두 유리합니다.

이 블로그도 `vite-react-ssg`로 SSG를 적용했습니다.
