# 사이트 런칭·운영 템플릿 (검색등록 · 방문통계 · 문체)

> 새 사이트를 낼 때 이 문서를 복사해 **[사이트별 설정값]** 만 바꾸고 체크리스트를 따라가면 됩니다.
> (Vite + React + SSG + GitHub Pages 기준. 다른 스택이면 "메타 삽입 위치"만 각자 방식으로.)
> ⚠️ 비밀키(서비스계정 JSON 등)는 이 문서에 적지 말 것 — GitHub Secrets 에만.

## 1. 검색엔진 등록 체크리스트

### 구글 Search Console
- [ ] search.google.com/search-console → **URL 접두어**로 `https://{도메인}/` 추가
- [ ] 소유권 인증 — **사이트에 GA4(gtag.js)가 이미 있으면 자동 인증됨** (인증코드 입력 불필요)
      - GA4 없으면: 'HTML 태그' 방식 → 코드 값을 `site.config.ts` `verification.google` 에 넣고 배포 후 [확인]
- [ ] 사이트맵 제출: `sitemap.xml`
- [ ] 홈 URL 검사 → 필요 시 '색인 생성 요청'
- [ ] 확인: 구글에 `site:{도메인}` 검색 → 색인 페이지 수
- ⚠️ 자동인증 유지하려면 GA4 코드 삭제 금지.

### 네이버 서치어드바이저
- [ ] searchadvisor.naver.com → 로그인 → 웹마스터도구 → 사이트 추가 `https://{도메인}/`
- [ ] 소유확인 → **HTML 태그** 선택 → `content="..."` 값 복사
- [ ] `site.config.ts` `verification.naver = '복사한값'` → 커밋·배포
- [ ] 라이브 반영 확인 후( 아래 확인 명령 ) 네이버에서 **[소유확인]** 클릭
- [ ] 사이트맵(`sitemap.xml`) + RSS(`feed.xml`) 제출, 수집 요청
- 참고: naver 서치어드바이저는 Claude 크롬 도구 접근 차단 → 네이버 화면 클릭은 사람이 직접, 코드/배포는 Claude.

메타 자동 삽입 배선(참고): `src/lib/head.tsx` 가 `verification.google/naver` 값이 있으면
`<meta name="google-site-verification" ...>` / `<meta name="naver-site-verification" ...>` 를 `<head>` 에 넣음.

## 2. 방문 통계 (사이드바 오늘/누적) — GitHub Actions + GA4 Data API

구조: 크론(`stats.yml`)이 GA4 Data API 조회 → `public/stats.json` 갱신·커밋 →
deploy 트리거 → 프론트(`Sidebar.tsx`)가 `/stats.json` 을 읽어 표시(값 없으면 자동 숨김).
관련 파일: `scripts/fetch-ga4-stats.mjs`, `.github/workflows/stats.yml`, `src/components/Sidebar.tsx`, 시드 `public/stats.json`.

### 설정 절차 (실제 검증된 순서)

**① Google Cloud (console.cloud.google.com)**
- [ ] **Analytics Data API 사용 설정**: API 라이브러리 → "Google Analytics Data API" → `사용`
      (직접 링크: `console.cloud.google.com/apis/library/analyticsdata.googleapis.com`)
- [ ] **서비스 계정 생성**: IAM 및 관리자 → 서비스 계정 → 만들기. 이름만 넣으면 이메일 자동 생성
      (예: `pancoit-stats@<프로젝트>.iam.gserviceaccount.com`) → **이 이메일 복사**
- [ ] **JSON 키 생성**: 만든 계정 → `키` 탭 → 키 추가 → 새 키 만들기 → **JSON** → 다운로드(=비밀키)

**② GA4 (analytics.google.com)**
- [ ] **속성 ID(숫자) 확인**: GA4 접속 시 URL 이 `#/a<계정ID>p<속성ID>/...` → **`p` 뒤 숫자가 속성 ID**
      (측정 ID `G-XXXX` 와 다름. 데이터 스트림 세부정보에서 측정 ID 일치 여부로 올바른 속성인지 교차 확인)
- [ ] **서비스 계정을 뷰어로 추가**: 관리 → 속성 → **속성 액세스 관리** → `+` → 사용자 추가 →
      위 서비스계정 이메일 붙여넣기 → **`이메일 알림` 체크 해제**(서비스계정은 메일 수신 불가) → 역할 **뷰어** → 추가

**③ GitHub (레포 → Settings → Secrets and variables → Actions)**
- [ ] `GA4_PROPERTY_ID` = 숫자 속성 ID
- [ ] `GA4_SA_KEY` = 다운로드한 JSON 키 **전체**(`{` 부터 `}` 까지 통째로)

**④ 실행**
- [ ] Actions → "Update visitor stats" → **Run workflow** 1회 → `stats.json` 실데이터로 갱신 → 배포 → 사이드바 노출

### ⚠️ 반드시 알아야 할 함정 2가지 (이번에 실제로 겪음)

1. **크론 push 는 배포를 자동 트리거하지 않는다.**
   워크플로가 `GITHUB_TOKEN` 으로 main 에 push 하면 `on: push` 인 deploy 가 **안 돈다**(GitHub 재귀 방지 규칙).
   → `stats.yml` 에 `permissions: actions: write` + 마지막 단계 `gh workflow run deploy.yml --ref main` 로 **명시 트리거**.

2. **빌드 가드(미래날짜 검증)는 날짜를 KST(+09:00)로 파싱해야 한다.**
   빌드는 러너=**UTC** 에서 도는데, 글 날짜(KST로 작성)를 러너 로컬로 파싱하면 KST 오전 시각이 UTC 기준 '미래'로 오판돼
   **배포가 전부 차단**된다. `scripts/validate-posts.mjs` 의 `toDate` 에서 `new Date(iso + '+09:00')` 로 파싱.

### 참고: "오늘" 이 0 으로 보이는 이유
GA4 **표준 Data API 는 실시간이 아니다** — 오늘 데이터는 집계 지연(수 시간)이 있어 실시간 화면에 방문자가 있어도
"오늘 활성 사용자"가 한동안 0/과소로 나온다(정상). 누적(totalUsers)은 정확. 즉시성이 필요하면 Realtime API(지난 30분 활성)로 대체 가능.

## 3. 문체 원칙 (교재형)

- 전체 **'니다체'** 사용, **'해요체' 금지**. 예: "…블로그입니다", "…배워 나갑니다".
- 예외(그대로 둠): 예제 **코드/출력**(예: `System.out.print("하세요")`, 출력 `안녕하세요`).
- 섹션 제목은 명사구 또는 "~란 무엇인가 / 어떻게 되는가" 로 통일.
- 점검: 소스에서 종결어미 `예요|에요|해요|어요|아요|나요|가요|까요|세요` 검색 → 코드/출력 제외하고 0 이어야 함.

## 4. 자주 쓰는 확인 명령

```bash
# 라이브에 메타(인증코드 등) 반영됐는지
curl -s "https://{도메인}/?cb=1" | grep -o 'naver-site-verification[^>]*'

# 로컬 == 원격 동기화 (0  0 이면 동일)
git rev-list --left-right --count origin/main...HEAD

# 프리렌더 결과에 남은 해요체 점검(코드/출력 제외)
grep -roiE '[가-힣]+(예요|에요|해요|어요|아요|나요|가요|까요|세요)' dist/*.html | grep -viE '안녕하세요|print'
```
