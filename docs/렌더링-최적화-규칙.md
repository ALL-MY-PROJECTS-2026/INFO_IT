# 렌더링 속도 최적화 규칙 (PANCO_IT)

목표 지표 — **Core Web Vitals** (실제 사용자 75 백분위 기준)

| 지표 | 의미 | Good |
|---|---|---|
| **LCP** | 가장 큰 콘텐츠 로딩 | ≤ 2.5초 |
| **INP** | 상호작용 반응성 | ≤ 200ms |
| **CLS** | 시각적 안정성 | ≤ 0.1 |

---

## 1. 이미 적용된 최적화 (건드리지 말 것)

- **SSG 프리렌더** — 각 라우트를 정적 HTML로 생성(`prerender.mjs`). 첫 화면이 JS 없이 바로 보임 → LCP/FCP 최적. (이게 가장 큰 레버)
- **Shiki 빌드타임 코드 하이라이팅** — 코드블록에 클라이언트 JS 0.
- **웹폰트 비차단 로드** — `index.html` 에서 Google Fonts CSS 를 `preload` + `media="print" onload` 로 로드해 **렌더 차단 제거**. `display=swap` 으로 텍스트 즉시 표시. 한글은 Google 의 `unicode-range` 서브셋이 효율적이라 그대로 사용.
- **react 벤더 청크 분리** — `vite.config.ts` 의 `manualChunks` 로 `react/react-dom/react-router-dom` 를 별도 청크로. 앱 코드가 바뀌어도 이 청크는 캐시 유지.
- **GA4 유휴(idle) 로드** — `src/lib/head.tsx` 에서 `gtag/config` 는 즉시 정의(이벤트 큐잉)하고 실제 `gtag.js` 는 `requestIdleCallback` 로 로드 → 초기 렌더 경로에서 제외.
- **자산 해시 캐싱** — Vite 가 파일명에 해시를 넣어 불변 캐싱(GitHub Pages 가 gzip/HTTP2 제공).
- **관리자 패널 dev 전용** — `import.meta.env.DEV` 로 감싸 프로덕션 번들에서 완전히 제외(트리셰이킹).
- **임베드/이미지 CLS 방지** — Looker iframe 은 높이 고정 + `lazy`. `img/svg/video` 는 `max-width:100%`.

---

## 2. 글·콘텐츠 작성 시 규칙 (이미지가 핵심)

- **포맷**: `AVIF > WebP > JPEG/PNG`. 가능하면 AVIF/WebP 로 저장.
- **크기 지정 필수**: 모든 이미지에 `width`/`height`(또는 `aspect-ratio`) → **CLS 방지**.
- **지연 로딩**: 첫 화면 **밖** 이미지는 `loading="lazy"`.
- **LCP 이미지(첫 화면 큰 이미지)**: `loading="eager"` + `fetchpriority="high"`, 필요하면 `<link rel="preload">`. **절대 lazy 걸지 말 것.**
- 업로드 이미지는 너무 크지 않게(가로 1600px 이하 권장), 용량 최적화 후 올리기.
- 본문 긴 URL·문자열은 자동 줄바꿈됨(`.prose { overflow-wrap: break-word }`).

---

## 3. 개발 시 규칙

- **JS 최소화(INP·TBT)**: 무거운 라이브러리 도입 자제. 꼭 필요하면 **동적 import(lazy)** 로 분리. 새 페이지/무거운 위젯은 `React.lazy` + `Suspense`.
- **서드파티 스크립트**: 되도록 async + 유휴 로드. 광고/분석/임베드는 초기 임계 경로에 넣지 않기. (Disqus 같은 무거운 위젯 지양 — 애드센스·속도·프라이버시에 불리)
- **폰트 추가 시**: `display=swap` 유지, 꼭 필요한 굵기/서브셋만. 새 폰트는 렌더 차단이 되지 않게 위 비차단 패턴 사용.
- **CLS**: 동적으로 삽입되는 요소(광고·배너·임베드)는 **자리(높이)를 미리 예약**.
- **애니메이션**: `transform`/`opacity` 위주(레이아웃 리플로우 유발 속성 지양), `prefers-reduced-motion` 대응(이미 전역 적용).
- **접근성 겸 성능**: 시맨틱 HTML, 불필요한 wrapper div 최소화.

---

## 4. 측정 방법

- 배포 후 **PageSpeed Insights**(https://pagespeed.web.dev) 에 `https://all-my-projects-2026.github.io/INFO_IT/` 입력 → 모바일/데스크톱 LCP·INP·CLS 확인.
- 크롬 **Lighthouse**(개발자도구 > Lighthouse) 로 로컬/라이브 점검.
- (선택) `web-vitals` 라이브러리로 실사용 지표를 GA4 로 전송해 필드 데이터 수집.

---

## 5. 향후 더 밀어붙일 수 있는 것 (필요 시)

- **폰트 self-host + 커스텀 서브셋**(fonttools/glyphhanger 로 실제 쓰는 글자만) → 한글 폰트 용량 대폭 감소. 콘텐츠 글자 범위가 안정되면 검토.
- **부분 하이드레이션/아일랜드**(Astro 등)로 전환 → 정적 콘텐츠에 JS 0. 큰 리아키텍처라 규모 커지면 검토.
- 커스텀 도메인 연결 시 `robots.txt`/사이트맵을 루트로 옮겨 SEO·크롤링 개선.
