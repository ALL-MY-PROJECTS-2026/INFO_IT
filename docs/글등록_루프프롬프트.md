# 글등록 루프 프롬프트 (PANCO_IT)

> 목적: PANCO_IT 블로그에 글 1편을 **작성 → 로컬 빌드 검증 → 커밋 → 푸시(자동 배포)**까지 안전하게 등록하기 위한 실행 절차.
> 사이트 정체성: **"어려운 IT를 초·중·고 누구나 이해하게, 그림과 비유로 쉽고 친근하게."** 모든 글이 이 톤을 지킵니다.
> 콘텐츠 품질·문체 원칙 원본: `SIDE_JOB/00_GITHUBPAGE_구글애드센스/글쓰기(IT교육).md`, `글쓰기(IT소식).md`, `글쓰기(프로젝트).md`.

---

## 0. 등록 파이프라인 (전체 흐름)

```
주제 선정 → src/content/posts/<slug>.mdx 작성 → npm run build (로컬 검증)
        → 프리렌더/사이트맵 반영 확인 → git add <그 mdx> → commit
        → git fetch + (필요시) rebase → git push origin main
        → GitHub Actions(deploy.yml)가 빌드 후 gh-pages 배포 → 라이브 반영
```

> 글 등록 = **`src/content/posts/`에 `.mdx` 파일 추가**입니다. 로컬 admin UI(`npm run dev` → `/admin`)로 넣어도 결국 같은 파일을 씁니다. 이 문서는 파일을 직접 추가하는 방식 기준입니다.

## 1. 파일명(slug) 규칙

- 위치: `src/content/posts/<slug>.mdx`
- slug: **소문자·숫자·하이픈만** (`^[a-z0-9](?:[a-z0-9-]*[a-z0-9])?$`). 한글·공백·대문자 금지.
- 예: `passkey-passwordless-login.mdx`, `what-is-http3.mdx`

## 2. frontmatter 규격 (정확히 지킬 것)

```mdx
---
title: 글 제목 (따옴표 없이, 콜론 포함 시 따옴표)
date: 2026-07-23
description: 검색·SNS 미리보기 요약 (150자 내외, 핵심 키워드 포함)
category: IT소식
tags: [보안, 인증]
---
```

- **category는 아래 5개 중 정확히 하나** (오타 시 분류 누락):
  | category | 용도 |
  |---|---|
  | `IT소식` | 따끈한 IT·개발 소식을 쉽게 |
  | `웹 개발` | 프론트·백엔드 기술 |
  | `웹 성능·SEO` | 빠르고 검색 잘 되는 사이트 |
  | `클라우드·인프라` | 배포·운영·자동화 |
  | `CS 기초` | 컴퓨터 동작 원리·개발 상식 |
- **미완성 글은 `draft: true`** 를 frontmatter에 추가 → 사이트맵·목록에서 제외됩니다(검수 후 제거하고 발행).
- 실제 글이 있는 카테고리만 사이드바·사이트맵에 노출됩니다. **소수 카테고리에 깊게 쌓는 것**을 권장.

## 3. MDX 작성 규칙 (빌드 안전 — 중요)

MDX는 마크다운을 JSX로 컴파일합니다. 잘못 쓰면 **빌드가 깨져 배포가 실패**하니 아래를 지킵니다.

- **순수 마크다운을 기본**으로: 제목(`##`), 문단, 표, 목록, 코드블록(```), 링크, 이미지.
- **원시 `<svg>`·HTML/JSX는 신중히.** 넣으려면 JSX 규칙(속성 문법)을 지켜야 하고, 반드시 로컬 빌드로 검증. 불확실하면 **SVG는 파일로 저장해 이미지로 삽입**(4장).
- 본문에서 `{`, `}`, `<` 를 그대로 쓰면 JSX로 해석될 수 있음 → 코드블록/백틱 안에 넣거나 피함.
- 코드 하이라이팅은 Shiki가 빌드 타임에 처리하므로, 코드블록에 **언어 지정**(```js 등) 권장.

## 4. 이미지 규칙

### 서두 대표 이미지 (무료 스톡 1장)
- **출처**: 무료·상업적 사용 OK 스톡만 — Unsplash·Pexels·Pixabay. 뉴스 사진·검색 이미지·핫링크 ❌.
- **관련성 최우선**: 글의 **핵심 소재를 직접 보여주는** 이미지. 검색 후 alt/설명을 읽고 글 내용과 맞는 것 선택.
  (예: 패스키 글 → 지문 스캐너 터치 손 / 랜섬웨어 → 잠긴 노트북·경고)
- **렌더링 최적화(필수)**: 가로형 크롭 + 압축해서 받는다. Pexels 예:
  ```bash
  curl -sL "https://images.pexels.com/photos/<ID>/pexels-photo-<ID>.jpeg?auto=compress&cs=tinysrgb&w=1200&h=630&fit=crop" \
    -o public/uploads/<slug>-hero.jpg
  ```
  폭 **1200px 내외 · 파일 ~100KB 이하** 목표(히어로 1200×630 ≈ 30~80KB).
- **삽입**: frontmatter 바로 아래 `![설명(alt)](/uploads/<파일명>.jpg)` + 다음 줄에 출처.
  ```mdx
  ![지문 스캐너에 손가락을 대는 모습](/uploads/passkey-hero.jpg)

  *이미지 출처: Pexels (pexels.com) · Pexels License*
  ```
  경로는 `/uploads/...` 그대로 쓴다 — `src/lib/mdxComponents.tsx`의 `withBase`가 **base(`/INFO_IT/`)를 자동 부착**하고 `loading="lazy"`도 적용한다(직접 base 붙이지 말 것).

### 본문 이해용 이미지 (자체 제작 · 디자인 시안은 "참고"만)
- **직접 새로 만든다.** 카테고리 디자인 키트(`SIDE_JOB/00_GITHUBPAGE_구글애드센스/디자인시안/<카테고리>/`)는 **스타일·구성 아이디어 참고용**이다.
  - ⚠️ **템플릿 SVG를 그대로 복붙하지 말 것.** 톤·형식만 참고해 **글 내용에 맞게 새로** 그린다(개념 도식·비유·숫자 요약 등).
  - 형식 아이디어: 비유 2분할 · 단계 플로우 · 개념 도식 · 오해vs사실 · 숫자 강조 · 용어 카드 · 장단점 · 타임라인 · 인용 배너.
- 자작 SVG를 `public/uploads/`에 저장 후 삽입. 원본이라 출처 불필요. 모든 이미지 `alt` 필수, 경량.
- ❌ 뉴스 기사 사진·구글/네이버 이미지 검색 결과·핫링크(저작권).

## 5. 콘텐츠 품질 규칙 (사이트 정체성)

- **니다체 통일**('~습니다/~입니다'), 초·중·고 눈높이. 전문 용어는 처음 나올 때 풀이.
- **비유로 직관 → 정확한 정의 → 예시**. "쉽게 —" 비유를 적극 사용.
- **원본성**: 남의 글 요약·재작성 금지. 비유·자작 도식·본인 정리로 유일하게.
- **완결성**: "그다음 궁금증"까지 답. **함정/자주 틀리는 것** 1개 이상.
- **정확성·시점**: 변동 정보는 "2026년 X월 기준" 명시. 미검증 수치·단정 금지.
- **한 장 정리**(표 + 핵심 3줄)로 마무리.
- **YMYL 주의**(금융·의료·보안): 정확성 강화, 개인 맞춤 조언 금지. 보안은 **방어 관점만**(악용법·PoC 금지).

## 6. 발행 전 로컬 검증 (필수 — 라이브 깨짐 방지)

```bash
npm run build
```

빌드 로그에서 확인:
- `✓ 프리렌더 완료: N 페이지` — **N이 이전보다 1 증가**했는가(새 글 반영)?
- `✓ sitemap.xml 생성 완료 (N URLs, 초안 제외)` — 글 포함(draft면 제외)?
- `✓ feed.xml 생성 완료` — RSS 반영?
- 에러 없이 완료됐는가? (MDX 오류 시 여기서 실패)

추가 확인(선택):
```bash
grep -l "핵심키워드" dist/posts/<slug>/index.html   # 본문이 정적 HTML에 렌더됐는지(CSR 아님)
grep -r "__admin/api" dist/                          # 결과 없어야 정상(관리자 코드 프로덕션 미포함)
```

## 7. 커밋 & 배포

```bash
git add src/content/posts/<slug>.mdx        # 그 글만(빌드 산출물 dist는 gitignore)
git commit -m "글: <제목> (<category>)"
git fetch origin
# origin/main이 앞서 있으면: git rebase origin/main  (충돌 없으면 자동)
git push origin main                         # → deploy.yml이 자동 빌드·배포
```

- 푸시 후 **GitHub Actions(deploy.yml)** 가 `npm ci && npm run build` 실행 → `dist/`를 `gh-pages` 브랜치로 배포. 보통 1~2분.
- 라이브 URL: `https://all-my-projects-2026.github.io/INFO_IT/posts/<slug>`
- ⚠️ 이 repo는 **다른 세션 워크트리**가 있을 수 있으니, 푸시 전 `git fetch` 후 분기 여부를 확인하고 필요 시 rebase로 통합(강제 푸시 금지).
