## 📌 0. 핵심 요약 (TL;DR)

1. **CRA는 2025-02-14 공식 지원 종료(deprecated).** ✅ 순수 CSR SPA는 SEO·애드센스에 가장 불리한 출발점. 최소한 **빌드 타임 프리렌더(SSG)** 필요.
2. **콘텐츠 블로그의 정공법은 Astro 또는 Next.js.** CRA식 Vite+React SPA를 고수하려면 반드시 `SSG + react-helmet-async + sitemap`을 얹어야 함.
3. **애드센스 공식 요건은 의외로 단순**: 만 18세 / 고품질·독창 콘텐츠 / 정책 준수 / 사이트 HTML 접근 가능. ✅ **최소 트래픽·글 수·도메인 나이·글자 수 강제 기준은 공식적으로 없음.** ⚠️ "글 20개, 800자, 도메인 6개월" 등은 커뮤니티 통념.
4. **개인정보처리방침 페이지는 실질적 필수** — 애드센스 심사 + 한국 개인정보보호법 제30조 양쪽에서 요구.
5. **AI 콘텐츠 자체는 금지 아님.** 사람의 검수·고유 가치·전문성(E-E-A-T)이 없는 대량 양산 콘텐츠가 제재 대상.
6. **2026 Core Web Vitals 임계값 변동 없음**: LCP ≤ 2.5s / INP ≤ 200ms / CLS ≤ 0.1 (75 백분위 기준). ✅

# Part 1. 2026 UI/UX 디자인 트렌드 (블로그 적용)

## 1-1. 우선순위별 트렌드

### 🔴 최우선 (기술 블로그 핵심 = 가독성 + 신뢰)

**1. 장문 가독성 최적화** ✅
- 줄 길이 50~75자(이상 66자, `max-width: 66ch`), 본문 16px 이상(장문은 18~20px 유리), 줄 높이 1.5.
- 여러 독립 출처에서 일관 확인. 기술 글은 체류 시간이 길어 가독성 저하 = 이탈 직결.

**2. 접근성 / WCAG 2.2 준수 (선택 아님)** ✅
- WCAG 2.2가 현재 기준선(2023-10 W3C 권고), EU 접근성법(EAA) 2025-06 시행.
- 본문 대비 4.5:1(큰 텍스트 3:1), 비텍스트 UI·포커스 3:1, 신규 AA: 포커스 외형(2.4.11), 타깃 크기 24×24 CSS px(2.5.8), 드래그 대안(2.5.7).

**3. 다크 모드 = 기본값** ✅ (개발자 독자층 선호율 특히 높음)
- 딥 차콜(순수 검정 X)·미드나잇 계열, 토큰화해 라이트/다크 대비 유지.

### 🟠 높음

**4. 목적 있는 마이크로 인터랙션 & 절제된 모션** ✅
- 화려함 → "목적 있는(purposeful)" 미묘한 모션. 지속 200~500ms, 물리 기반 이징, `prefers-reduced-motion` 대응 표준.

**5. 표현적 타이포그래피 (헤더는 개성, 본문은 절제)** ✅
- 큰 헤드라인·가변 폰트(variable fonts)는 히어로/헤더에, 본문은 위 1번 원칙 유지 (이원화).

**6. 벤토 그리드(Bento Grid) 레이아웃** ✅ 방향성 / ⚠️ 성과 수치
- Apple·Linear 주도. 일관 간격(16/24px), 셀 내부 여백 20~30%, 라운드 코너. 홈·카테고리·관련 글 배치에 유용.

**7. 장문 탐색 보조: 읽기 진행바 · 스티키 목차(TOC) · 예상 읽기 시간** ✅
- 스크롤 진행바, IntersectionObserver 기반 스티키 TOC, 읽기 시간 표시.

### 🟡 중간

**8. AI 기반 개인화 / 생성형 UI** — 관련 글 추천, 태그 개인화, AI 요약(TL;DR)/시맨틱 검색으로 현실 적용.
**9. "의도된 미니멀리즘"으로 수렴** ✅ — 기술·데이터 중심 산업엔 미니멀리즘이 최적. 절제된 기반 + 헤더/색 포인트만 대담하게.
**10. 색상: 시스템·토큰 중심** — 부드러운 그라디언트 + 다크 표면 위 마이크로 네온 액센트(CTA/포커스/배지). (Pantone 2026 = Cloud Dancer, 소프트 화이트)

## 1-2. ✅ 적용 체크리스트 (React 블로그)

**타이포그래피 · 가독성**
- [ ] 본문 컨테이너 `max-width: 65~70ch`, `margin-inline: auto`
- [ ] 본문 데스크톱 18~20px(최소 16px), 모바일 15~16px, `line-height: 1.5~1.6`
- [ ] 유동 타이포 `font-size: clamp(1rem, 0.9rem + 0.5vw, 1.25rem)`, rem/em 사용(200% 확대 대응)
- [ ] 본문 좌측 정렬(양쪽 정렬 금지), 코드 monospace

**다크 모드 · 색상 (토큰 기반)**
- [ ] CSS 변수/디자인 토큰 → `prefers-color-scheme` + 수동 토글
- [ ] 다크 배경 딥 차콜(#0d~#1a 계열), 라이트/다크 각각 대비 검증
- [ ] 액센트는 CTA·링크·포커스에만, 토글 상태 localStorage 저장 + FOUC 방지 초기 스크립트

**접근성 (WCAG 2.2 AA)**
- [ ] 본문 대비 ≥ 4.5:1, 큰 텍스트/비텍스트 UI ≥ 3:1 (라이트·다크 각각)
- [ ] 커스텀 포커스 링 `:focus-visible`(대비 ≥ 3:1)
- [ ] 클릭 타깃 ≥ 24×24px(모바일 44px 권장)
- [ ] `prefers-reduced-motion: reduce` 대응, 색만으로 정보 전달 금지, 시맨틱 HTML + 키보드 내비게이션

**레이아웃 · 장문 UX**
- [ ] 홈/카테고리/관련 글 CSS Grid 벤토 스타일, 모바일 단일 컬럼 리플로우
- [ ] 상단 고정 읽기 진행바(`requestAnimationFrame`)
- [ ] 스티키 TOC(`IntersectionObserver`로 현재 섹션 하이라이트), 모바일 접이식
- [ ] 예상 읽기 시간 + 발행/수정일 상단 표기
- [ ] 코드 블록: 구문 강조 + 복사 버튼 + 언어 배지 + `overflow-x: auto`

**모션**
- [ ] 인터랙션 200~500ms, 물리 기반 이징, 상태 변화에만 피드백(장식성 자제)

## 2-2. 정보구조 · 핵심 기능
- **계층형 + 매트릭스 혼합 IA**, 브레드크럼(구조화 데이터 병행), 글로벌 내비게이션 + 내부 링크, 의미 있는 URL(`/category/post-slug`)
- **검색**(SPA는 Lunr/FlexSearch 클라이언트 인덱스 또는 Algolia/Typesense), **카테고리·태그**, **댓글**(Giscus/Disqus/Utterances), **RSS/Atom**, **소셜 공유/OG**, **다크 모드**, **코드 하이라이팅·복사**, **목차**, **애널리틱스**(GA4 + Search Console)

## 2-3. ✅ Core Web Vitals 목표치 (2026 — 변동 없음, 3회차 재검증)

Google은 CrUX **75 백분위수** 기준 평가(방문자 75% 이상이 "Good").

| 지표 | Good | 개선 필요 | Poor |
|---|---|---|---|
| **LCP** (로딩) | **≤ 2.5s** | 2.5–4.0s | > 4.0s |
| **INP** (반응성, FID 대체) | **≤ 200ms** | 200–500ms | > 500ms |
| **CLS** (시각 안정성) | **≤ 0.1** | 0.1–0.25 | > 0.25 |

## 2-4. ✅ SEO 체크리스트
- [ ] 페이지별 고유 `<title>`(50~60자), `meta description`(150~160자), `canonical`
- [ ] **JSON-LD 구조화 데이터**: 블로그 글 = `BlogPosting`, 필수 속성 `headline`/`author`/`datePublished`/`dateModified`/`image`(폭 ≥ 1200px). Google은 Microdata 폐기, **JSON-LD 권장**
- [ ] 추가 스키마: `BreadcrumbList`, `WebSite`, `Person`/`Organization`
- [ ] `sitemap.xml` 제출 + `robots.txt` 설정
- [ ] Open Graph + Twitter Card (og:title/description/image)
- [ ] Rich Results Test·URL Inspection 검증
- [ ] 301/302 올바른 사용, 깨진 링크 정리
- [ ] 모바일 우선 색인 대응(반응형), 콘텐츠 수정 시 `dateModified` 갱신

## 2-5. ✅ 성능 최적화 체크리스트
- [ ] **LCP**: above-the-fold 대표 이미지는 **lazy-load 금지 + `preload`** (대표적 실수 주의)
- [ ] **이미지**: AVIF > WebP > JPEG (`<picture>` 폴백), `srcset`/`sizes` 반응형, 스크린 밖은 `loading="lazy"`, **width/height 지정으로 CLS 방지**
- [ ] **CLS**: 이미지·광고·임베드 크기 예약, 웹폰트 `font-display`
- [ ] **INP**: JS 축소, 코드 스플리팅, 메인 스레드 작업 분할
- [ ] CDN·캐싱·압축(gzip/brotli)

## 2-6. ⚠️ CRA의 SEO 한계와 해결책 (중요)
- **CRA = CSR SPA** → 초기 HTML이 비어 있고 콘텐츠가 JS로 렌더. Googlebot이 렌더링은 하나 **2차 색인 큐로 밀려 수일~수주 지연** 또는 타임아웃/실패 가능.
- CRA는 라우팅·데이터 페칭·코드 스플리팅·SSR/SSG 미제공.
- **해결책(강도순)**:
  1. `react-helmet-async`로 라우트별 메타/OG 주입 (최소 대응, 근본 해결 아님)
  2. 프리렌더: `react-snap`(⚠️ 유지보수 중단 관측) / Prerender.io 미들웨어
  3. **프레임워크 이전(권장): Next.js / React Router 프레임워크 모드**
  4. SSG 대안: Astro, Hugo, Eleventy

# Part 3. 실전 구현 스택 (React + TypeScript)

## 3-1. 프레임워크 비교 (블로그 + SEO + AdSense 기준)

| 항목 | Vite+React SPA (+SSG) | Next.js | Astro (+React 아일랜드) |
|---|---|---|---|
| 렌더링 기본 | CSR (SSG는 추가) | SSR/SSG/ISR 내장 | **정적 HTML 기본, JS ~0KB** |
| SEO 적합성 | 프리렌더 필수 | 우수 | **최상** |
| AdSense/광고 | 손 많이 감 | 무난 | 무난(광고는 아일랜드) |
| 러닝커브 | 라이브러리 직접 조합 | 학습곡선·생태계 최대 | 콘텐츠 쉬움, 강인터랙션 불리 |
| 배포 | 정적 → CDN | 서버/엣지(정적 export 가능) | 정적 → CDN |

**추천**:
1. **콘텐츠 블로그 정공법 → Astro + React 아일랜드** (SEO/성능 최상, React/TS 지식 재활용)
2. **React 풀스택·앱 확장성 → Next.js (App Router)**
3. **CRA식 Vite+React SPA 고수 시 → 반드시 SSG 프리렌더 필수**

> ⚠️ "블로그의 80%는 Astro"·"Vite가 더 낫다"는 각 출처의 **의견**(이해관계 반영). 반면 CRA deprecation·번들 크기 차이·SSG 필요성은 **다중 검증된 사실**.

## 3-2. Vite + React + TS 블로그 라이브러리 스택

| 역할 | 라이브러리 | 비고 |
|---|---|---|
| 빌드 | **Vite** + `@vitejs/plugin-react` | CRA 대체 |
| **프리렌더(핵심)** | **vite-react-ssg**(RR v6) 또는 **React Router v7 내장 SSG** | 빌드 타임에 각 글 정적 HTML화 → SEO/AdSense 전제 |
| (대안) | **Vike**(구 vite-plugin-ssr) | SSG↔SSR 유연 |
| 라우팅 | **react-router** v6/v7 | 글마다 고유·크롤 가능 URL 필수 |
| 콘텐츠 | **MDX** (`@mdx-js/rollup`) | 마크다운 + JSX |
| 메타/SEO | **react-helmet-async** | 페이지별 title/OG/canonical, 빌드 타임에 HTML화 |
| 코드 하이라이팅 | **Shiki**(권장, 빌드 타임) 또는 Prism | 클라이언트 JS 부담 적음 |
| 마크다운 플러그인 | remark/rehype (`rehype-slug`, `rehype-autolink-headings`, `remark-gfm`) | 목차 앵커·GFM |
| 사이트맵/RSS | vite-plugin-sitemap 또는 빌드 스크립트 | Search Console 제출 |
| 크리티컬 CSS | beasties (vite-react-ssg 내장) | LCP/FOUC 개선 |
| 스타일 | Tailwind CSS 또는 CSS Modules | 취향 |
| 구조화 데이터 | JSON-LD (`BlogPosting`) 직접 주입 | 리치 결과 |

**최소 필수 세트**: `Vite + React + TS → react-router → MDX → react-helmet-async → 프리렌더 → sitemap`
(프리렌더·sitemap 빼면 AdSense·SEO 크게 불리)

## 3-3. ⚠️ AdSense + SPA 통합 주의점
1. `adsbygoogle.js`는 `index.html`에 **1회만** 로드
2. 광고 위치마다 `<ins class="adsbygoogle">` 배치
3. `(window.adsbygoogle=window.adsbygoogle||[]).push({})`는 **`<ins>` 렌더 이후** 호출
4. SPA는 **라우트 변경마다 push 재실행** 필요

**흔한 함정**:
- **"already have ads" 오류**: 같은 슬롯 중복 push (StrictMode 이중 렌더·라우트 재진입 시). → 광고 컨테이너를 `key`로 언마운트/리마운트
- push 타이밍: `<ins>` DOM 존재 후 실행
- **CSR-only는 승인 리스크** → 프리렌더로 실제 콘텐츠 HTML 서빙
- **인위적 광고 리프레시는 역효과**(수익 저하 + 정책 위반 소지)
- 고유·크롤 가능 URL 필수 (해시 라우팅 지양, History API + SPA fallback)

# Part 4. Google AdSense 승인 (2025~2026)

## 4-1. ✅ 공식 자격 요건 (Google 문서 기준)

| 항목 | 공식 요건 |
|---|---|
| **최소 연령** | **만 18세 이상** (미만은 보호자 명의) |
| **콘텐츠** | 고품질(high-quality)·독창(original)·방문자를 끄는 콘텐츠 |
| **정책** | 애드센스 프로그램 정책 준수 |
| **사이트 접근** | HTML 소스에 광고 코드 삽입 가능 |
| **계정** | 유효한 Google 계정 |

**✅ 공식적으로 요구되지 않는 것 (중요 — 통념 주의)**:
- ❌ 최소 트래픽/페이지뷰 기준 없음
- ❌ 최소 게시물 수 기준 없음
- ❌ 최소 도메인 나이 규정 없음
- ❌ 최소 글자 수 규정 없음

> ⚠️ 다수 SEO 블로그가 "글 15~30개, 800~1000자, 도메인 6개월"을 요구조건처럼 제시하나(3회차 검증에서도 재확인), 이는 **커뮤니티 통념·모범 사례**이며 Google 공식 정책 로그에 명문화된 신규 승인 문턱 상향은 없음. 체감 강화는 **심사·집행 강화**에서 비롯.

## 4-2. ✅ 필수 페이지 체크리스트
- [ ] **개인정보처리방침**(광고·쿠키 데이터 수집 고지 → 사실상 필수)
- [ ] **소개(About)**(E-E-A-T, 심사 확인)
- [ ] **문의(Contact)**(실제 연락 수단, 심사 확인)
- [ ] 이용약관(커뮤니티 강력 권장)
- [ ] 메인 내비게이션에서 위 페이지 접근 용이

## 4-3. 콘텐츠 품질 (공식 vs 커뮤니티)
- **공식**: 고품질·독창, 충분한 가치 + 좋은 UX·탐색 요소, **E-E-A-T**(경험·전문성·권위·신뢰; 특히 Trust)
- ⚠️ **커뮤니티 권장**: 글 15~30개(자주 20~25), 글당 700~1200자, 표절·미편집 AI 금지
- **기술 블로그 특화**: 직접 찍은 스크린샷·실험·코드·벤치마크(경험 신호), 저자 바이오, 출처 인용
- **핵심 원칙: 양보다 질** (얇은 300자 글 50개 < 깊이 있는 글 15개)

## 4-4. 주요 거절 사유 & 회피
1. **Low value content(가장 흔함)**: 얇은 글 보강(700자+)·삭제, 독창 인사이트·E-E-A-T 강화
2. **정책 위반**: 성인·저작권·폭력·혐오·불법 제거, Policy Center 확인
3. **사이트 접근 불가**: 공개 상태 확인, 로그인 보호 해제, robots.txt에서 AdSense 크롤러 미차단, HTTPS
4. **광고 코드 누락**: `<head>`에 정확 삽입, 도메인 일치
5. **추가 실수**: 너무 이른 신청 / 쿠키 동의 도구가 광고 스크립트 차단 / 즉시 재신청(2~4주 대기 권장)

## 4-5. 🇰🇷 한국 시장 특화
- **네이버 블로그: 애드센스 불가**(외부 스크립트 차단, 애드포스트만). **티스토리/워드프레스/블로거: 가능.**
- ⚠️ **커스텀 도메인은 필수 아님·권장**(티스토리 서브도메인 승인 사례 있으나 개인 도메인이 신뢰도·승인률 유리)
- ⚠️ 한국 커뮤니티 후기: 승인 기간 대체로 **2일~2주**(최대 1개월), 글 수는 **10~30편** 편차 큼, 권장선 "1000자+ 고품질 10~15편"
- 심사 중에도 **꾸준히 발행** 권장

**PIPA(개인정보보호법 제30조) — 개인정보처리방침 필수 포함 8항목**:
1. 수집 항목(이메일·이름·**쿠키**·IP) 2. 수집·이용 목적 3. 보유 기간 4. **제3자 제공(Google)** 5. 처리 위탁 6. 정보주체 권리 7. 보호책임자 연락처 8. **쿠키·추적기 안내(GA·AdSense 명시)**
- 실제 수집 실태와 일치, 푸터/전용 페이지 상시 노출 + 개정일 표기
- ⚠️ 쿠키 동의: 한국은 EU식 사전동의 법적 명문 의무는 아니나 '수락/거부' 선택권 제공이 권장. **단, EU/EEA·영국 트래픽이 있으면 Google EU 사용자 동의 정책상 CMP 필요**

## 4-6. AI 콘텐츠에 대한 Google 입장 (2026)
- **"AI 금지" 규정 없음.** 제작 방식이 아닌 **결과물 품질**로 판단. 핵심 제재 개념 = **Scaled Content Abuse(대량 생성 남용)**
- **제재 대상**: 편집 없는 대량 발행, 구조 획일성, 검증 불가 일반론, 저자·바이라인 부재, 단순 AI 리라이트
- **통과 조건**: AI는 초안 도구로만, 실명 저자 + 검증 가능 배경, 원본 데이터·사례·관점, **다른 곳에 없는 인사이트 최소 1개**

## 4-7. ✅ 신청 전 Go / No-Go 체크리스트

**Go (모두 충족 시 신청 권장)**:
- [ ] 만 18세 이상 / 사이트 HTML 코드 삽입 가능
- [ ] HTTPS 적용 + 정상 공개, robots.txt 크롤러 미차단, 로그인 보호 없음
- [ ] 개인정보처리방침 + About + Contact (+ 이용약관 권장)
- [ ] 메뉴에서 페이지·콘텐츠 탐색 용이
- [ ] 독창·충실한 글 다수(권장 15개+), 각 글 실질 정보
- [ ] 표절/미편집 AI 없음, E-E-A-T 신호 포함
- [ ] 정책 위반 콘텐츠 없음
- [ ] (권장) `ads.txt` 루트 배치 (필수 아님, 강력 권장)

**No-Go (보류)**: 얇은 글 몇 개 / "준비 중" / 복붙·스크래핑 / 필수 페이지 누락 / 깨진 링크·미완성 / 정책 위반 / HTTPS 미적용

# 🗺️ 종합 실행 로드맵 (제안)

1. **스택 결정**: 콘텐츠 중심이면 **Astro** 또는 **Next.js** 강력 권장. CRA/Vite SPA 고수 시 SSG 프리렌더 필수 세트 구성.
2. **디자인 시스템**: 토큰 기반 색상(라이트/다크), 66ch 본문, WCAG 2.2 AA, 벤토 홈, 스티키 TOC + 읽기 진행바.
3. **필수 페이지 선구축**: About / Contact / 개인정보처리방침(PIPA 8항목) / 이용약관.
4. **SEO 인프라**: JSON-LD(`BlogPosting`), sitemap.xml, robots.txt, OG, canonical, GA4 + Search Console.
5. **콘텐츠 축적**: 깊이 있는 IT 글 15편+ (1000자+, 직접 경험·코드·스크린샷·출처, 저자 바이오). 꾸준한 발행.
6. **성능 검증**: LCP ≤2.5s / INP ≤200ms / CLS ≤0.1, 이미지 AVIF/WebP + width/height.
7. **AdSense 신청**: Go 체크리스트 충족 확인 → `<head>` 코드 삽입 → 심사(2~14일). 거절 시 원인 수정 후 2~4주 뒤 재신청.
8. **광고 통합**: `AdUnit` 컴포넌트로 중복 push 방지, 인위적 리프레시 금지, ads.txt 배치.
