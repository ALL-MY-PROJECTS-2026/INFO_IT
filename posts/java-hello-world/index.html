자바 첫 프로그램 — HelloWorld 찍어보기

Jungwoo2026년 7월 23일
#자바#JAVA#프로그래밍입문#HelloWorld

이미지 출처: Pexels · Pexels License

프로그래밍을 처음 배울 때 누구나 만드는 첫 프로그램이 있습니다. 바로 화면에 "Hello, World!" 한 줄을 찍는 것입니다.

별것 아닌 것 같지만, 이 짧은 코드 안에 자바의 뼈대가 다 들어 있습니다. 오늘은 이 첫 프로그램을 한 줄도 안 빼놓고 쉽게 뜯어보겠습니다. 지하철에서 5분이면 충분합니다.

프로그램이 대체 뭔가요?

프로그램은 컴퓨터에게 시키는 일의 순서입니다.

쉽게 — 프로그램은 요리 레시피와 같습니다. "① 물을 끓인다 ② 면을 넣는다 ③ 3분 뒤 건진다"처럼, 컴퓨터가 위에서 아래로 순서대로 따라 하도록 적어 둔 것이 프로그램입니다. 다만 컴퓨터는 눈치가 없어서, 정해진 문법으로 아주 정확하게 적어야 알아듣습니다.

그 문법 중 하나가 자바(Java)입니다.

첫 코드, 통째로 보기

먼저 완성된 모습부터 봅니다. 이게 전부입니다.

public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

실행하면 화면에 이렇게 나옵니다.

Hello, World!

짧습니다. 이제 이 네 줄을 하나씩 풀어 보겠습니다.

한 줄씩 뜯어보기

① public class HelloWorld { 자바는 모든 코드를 클래스(class)라는 상자 안에 담습니다. 상자 이름이 HelloWorld입니다.

쉽게 — 클래스는 공책의 표지라고 생각하면 됩니다. 표지에 제목(HelloWorld)을 적고, 그 안쪽에 내용을 채우는 것입니다. 여는 중괄호 {부터 맨 아래 닫는 }까지가 이 공책의 속입니다.

② public static void main(String[] args) { main은 프로그램이 시작되는 지점입니다.

쉽게 — main은 책의 1페이지입니다. 컴퓨터는 프로그램을 켜면 무조건 main부터 찾아 읽기 시작합니다. main이 없으면 "어디부터 읽어야 하지?" 하고 멈춰 버립니다.

지금은 public static void ... 같은 앞부분은 "main을 쓰려면 따라오는 주문" 정도로 외워 두면 됩니다. 뜻은 뒤 편에서 하나씩 배웁니다.

③ System.out.println("Hello, World!"); 이 한 줄이 진짜 일을 하는 부분입니다. 화면에 글자를 찍습니다.

  • System.out — 화면(모니터)을 가리킵니다.
  • println — "한 줄 출력하고 줄 바꿈"이라는 뜻입니다(print + line).
  • "Hello, World!" — 큰따옴표 안의 글자가 그대로 화면에 나옵니다.

④ } } 열었던 중괄호를 닫아 상자를 마무리합니다. main의 } 하나, class의 } 하나입니다.

실행은 어떻게 되나요?

내가 쓴 자바 코드는 사람 눈에는 읽히지만, 컴퓨터는 바로 이해하지 못합니다. 번역 단계를 거칩니다.

쉽게 — 한국어로 쓴 편지를 외국 친구에게 보내려면 영어로 번역해서 부치는 것과 같습니다. 작성(.java) → 컴파일(번역) → 실행, 이 3단계입니다. 컴파일이 "번역" 단계입니다.

  • 작성: HelloWorld.java 파일에 코드를 씁니다.
  • 컴파일: 컴퓨터가 알아듣는 형태(.class)로 번역합니다.
  • 실행: 번역된 것을 돌려 화면에 결과가 나옵니다.

지금은 이클립스(Eclipse)나 인텔리제이 같은 도구가 이 과정을 버튼 하나로 대신 해 줍니다.

자주 틀리는 곳 (여기서 다들 막힙니다)

단골 함정 4가지

① 세미콜론 ; 빼먹기 — 자바에서 문장 끝에는 반드시 ;를 붙입니다. println(...) 뒤에 ;가 없으면 에러가 납니다. (문장의 마침표라고 생각하십시오.)

② 대소문자 틀리기 — System을 system으로 쓰면 안 됩니다. 자바는 대문자와 소문자를 다른 글자로 봅니다.

③ 클래스 이름과 파일 이름 불일치 — public class HelloWorld라면 파일 이름도 반드시 **HelloWorld.java**여야 합니다. 이름이 다르면 실행되지 않습니다.

④ 큰따옴표 빠뜨리기 — 화면에 글자로 찍고 싶으면 "..." 안에 넣어야 합니다. 따옴표가 없으면 컴퓨터는 그걸 명령어로 착각합니다.

조금 바꿔보기

println 대신 print를 쓰면 줄을 바꾸지 않습니다. 차이를 봅니다.

System.out.print("안녕");
System.out.print("하세요");

결과: 안녕하세요 (한 줄에 붙음)

System.out.println("안녕");
System.out.println("하세요");

결과:

안녕
하세요

ln이 붙으면 "출력 후 엔터"라고 기억하면 됩니다.

한 장 정리

조각뜻
class코드를 담는 상자(공책 표지)
main프로그램이 시작하는 곳(1페이지)
System.out.println("…")화면에 한 줄 찍고 줄 바꿈
;문장의 마침표(빼먹으면 에러)

이것만 기억하면 됩니다. ① 자바 코드는 class 상자 안에 담고, 컴퓨터는 main부터 읽습니다. ② 화면 출력은 System.out.println("…")입니다. ③ 문장 끝 ;, 대소문자, 클래스명과 파일명 일치 — 이 셋만 조심하면 첫 에러의 90%는 피합니다.
