Core Web Vitals 2026 — LCP, INP, CLS 한눈에 정리

Jungwoo2026년 7월 18일
#성능#SEO#CoreWebVitals

Core Web Vitals는 구글이 **실제 사용자 데이터(CrUX)**로 페이지 경험을 평가하는 세 가지 지표입니다. 방문자의 75% 이상이 "Good"이어야 통과입니다.

세 가지 지표와 목표치

지표의미Good
LCP가장 큰 콘텐츠 로딩≤ 2.5초
INP상호작용 반응성≤ 200ms
CLS시각적 안정성≤ 0.1

이 임계값은 2026년에도 변경되지 않았습니다.

LCP 최적화

가장 흔한 실수는 대표 이미지를 lazy-load 하는 것입니다. 화면 상단 이미지는 오히려 미리 불러와야 합니다.

<!-- 상단 히어로 이미지: lazy 금지, preload 사용 -->
<link rel="preload" as="image" href="/hero.avif" />
<img src="/hero.avif" alt="..." fetchpriority="high" />

CLS 최적화

이미지·광고 영역에 크기를 미리 예약하면 레이아웃이 밀리지 않습니다.

<img src="/thumb.webp" width="800" height="450" alt="..." />

INP 최적화

  • 자바스크립트 번들을 줄이고 코드 스플리팅
  • 무거운 작업을 작은 단위로 분할
  • 정적 렌더링(SSG)으로 초기 JS 전송량 감소

정리

성능은 SEO 순위 요소이자 사용자 경험의 핵심입니다. 이미지 최적화 + 레이아웃 크기 예약 + JS 다이어트 세 가지만 지켜도 대부분의 블로그는 통과 기준에 도달합니다.
